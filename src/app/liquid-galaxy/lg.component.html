Liquid Galaxy

Memories supports Liquid Galaxy*. You can cast your journeys or stories on a Liquid Galaxy setup just like if it was a Chromecast device, by clicking on the icon.

Have a Liquid Galaxy ready? Jump to cast.

* Liquid Galaxy API required.

Install Liquid Galaxy

If you do not have a Liquid Galaxy setup yet, getting one ready just takes a few minutes (and all its code is open-source).

Grab a few average machines, connect each of them to a distinct screen, and follow these instructions.

  1. Install Ubuntu to each of each (see README for the current supported versions).
  2. Execute bash <(curl -s https://raw.githubusercontent.com/LiquidGalaxyLAB/liquid-galaxy/master/install.sh)

See detailed instructions here.

Install Liquid Galaxy API on the master machine

Memories does its cast magic by making use of a public API placed on the Liquid Galaxy master machine. Installing it is as simple as running the following script:

bash <(curl -s https://raw.githubusercontent.com/LiquidGalaxyLAB/liquid-galaxy-api/master/scripts/install.sh)

See detailed instructions here.

Cast

When your Liquid Galaxy is displaying Google Earth across its screens, and the API is up and ready (you can verify so by entering <ip-address>:82 on your browser), you are ready to cast.

Press the icon at the top, select your Liquid Galaxy setup1, and journeys and stories will start showing2 on the screens as you enter them on your device.

Liquid Galaxy users will only have to go through this step.

1 By default, Liquid Galaxy setups can only be discovered by those who are in the same network (so you might have to connect through Wi-Fi on your mobile devices). 2 Only geolocalized stories will be shown.
